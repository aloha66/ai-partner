# Project Guidance

## Skill routing

When a user request matches an available gstack skill, invoke that skill instead of improvising.

Suggested routing:
- Product ideas, framing, or scope questions -> `/office-hours`
- Architecture or implementation planning -> `/plan-eng-review`
- UI or interaction design review -> `/plan-design-review`
- Developer experience or onboarding questions -> `/plan-devex-review`
- Bug investigation or root-cause analysis -> `/investigate`
- Code review or production risk review -> `/review`
- QA, browser checks, or visual verification -> `/qa`
- Security or threat modeling -> `/cso`
- Release prep, shipping, or PR completion -> `/ship`

If you are unsure, prefer the gstack skill that best matches the user's intent before taking action.
